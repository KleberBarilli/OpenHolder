## Codebase Patterns
- `Holder.AIScoring.providers/0` returns the `@providers` map — use it for dynamic provider registration and Settings validation
- Use `Portfolio.get_or_create_default_portfolio()` + `ensure_default_classes(portfolio.id)` in test setup blocks
- The unique constraint on asset_classes is `[:portfolio_id, :key]` — errors appear on `:portfolio_id`, not `:key`
- `list_assets/2` and `get_asset!/1` auto-preload `:asset_scores`
- `mix precommit` runs compile --warnings-as-errors, deps.unlock --unused, format, test — pre-existing warnings in dashboard_live.ex, settings_live.ex, hooks.ex, quote_server.ex cause failure
- Pre-existing failing test: `HolderWeb.PageControllerTest` (home page text assertion mismatch)
- Use `errors_on(changeset)` from DataCase to test changeset validation errors
- Macro targets are stored as fractional values (0.25 = 25%), not percentages
- `export_json/1` iterates a hardcoded list of 7 class keys, not dynamic from DB
- `import_json/2` does full replace per class (deletes then recreates), not merge
- Vault and Prompt tests don't need DataCase — use `ExUnit.Case, async: true` for speed
- LiveView tests: `use HolderWeb.ConnCase` + `import Phoenix.LiveViewTest` — on_mount hook auto-creates portfolio
- Dashboard charts only render when `@summary.grand_total > 0` — don't assert canvas on empty portfolios
- `validate_response/2` in AIScoring is private — cannot unit test directly
- Dense table: non-RF has 11 columns, RF has 8 — update colspan on inline edit rows when adding/removing columns
- Use `hidden md:table-cell` for responsive table column visibility (Tailwind v4)
- ClassDetailLive uses `handle_params/3` + `load_class/2` for tab switching — `@all_classes` is loaded once in mount
- Inline edit inputs use `.inline-edit` CSS class (transparent bg, shows border on hover/focus) + `.InlineEdit` colocated JS hook for save-ok/save-err feedback
- `phx-blur` on `<input>` sends `%{"value" => el.value}` along with `phx-value-*` attributes — no extra JS needed for value extraction
